
### Architecture of Web & HTTP (Questions 1-8)

**1. Which HTTP method is idempotent and safe for retrieving resources?**
- a) POST
- b) PUT
- c) GET
- d) DELETE

**Answer: c) GET**

---

**2. What is the primary difference between HTTP/1.0 and HTTP/1.1?**
- a) HTTP/1.1 supports persistent connections
- b) HTTP/1.0 has better security
- c) HTTP/1.1 doesn't support GET method
- d) HTTP/1.0 supports multiplexing

**Answer: a) HTTP/1.1 supports persistent connections**

---

**3. Which HTTP status code indicates "Resource Not Found"?**
- a) 200
- b) 301
- c) 404
- d) 500

**Answer: c) 404**

---

**4. What does the stateless nature of HTTP protocol mean?**
- a) Server maintains client information across requests
- b) Each request is independent and contains all necessary information
- c) Client cannot send multiple requests
- d) Server stores session data permanently

**Answer: b) Each request is independent and contains all necessary information**

---

**5. Which component resolves domain names to IP addresses?**
- a) HTTP Server
- b) Domain Name Service (DNS) Server
- c) Web Browser
- d) Application Server

**Answer: b) Domain Name Service (DNS) Server**

---

**6. What is the primary purpose of HTTPS?**
- a) Faster data transfer
- b) Secure encrypted communication
- c) Larger file uploads
- d) Better caching

**Answer: b) Secure encrypted communication**

---

**7. Which HTTP method is used to update an existing resource?**
- a) GET
- b) POST
- c) PUT
- d) HEAD

**Answer: c) PUT**

---

**8. Which web server is NOT mentioned in the syllabus?**
- a) Apache
- b) IIS
- c) Nginx
- d) Both a and b are mentioned

**Answer: c) Nginx**

### React & React-Redux (Questions 45-50)

**45. What are the two main types of components in React?**
- a) Class and Object Components
- b) Function and Class Components
- c) Static and Dynamic Components
- d) Parent and Child Components

**Answer: b) Function and Class Components**

---

**46. What is the primary purpose of 'state' in React?**
- a) To style components
- b) To manage component data that can change
- c) To import libraries
- d) To define routes

**Answer: b) To manage component data that can change**

---

**47. What is the difference between props and state in React?**
- a) Props are mutable, state is immutable
- b) Props are passed from parent, state is managed within component
- c) No difference
- d) State is passed from parent, props are internal

**Answer: b) Props are passed from parent, state is managed within component**

---

**48. What are the three core concepts of Redux?**
- a) Components, Props, State
- b) Actions, Reducers, Stores
- c) Views, Models, Controllers
- d) Routes, Middleware, Templates

**Answer: b) Actions, Reducers, Stores**

---

**49. What is the purpose of Error Boundaries in React?**
- a) To prevent syntax errors
- b) To catch JavaScript errors in component tree and display fallback UI
- c) To validate forms
- d) To handle HTTP errors

**Answer: b) To catch JavaScript errors in component tree and display fallback UI**

---

**50. What is 'lifting state up' in React?**
- a) Moving state to a common ancestor component
- b) Increasing state value
- c) Deleting state
- d) Converting state to props

**Answer: a) Moving state to a common ancestor component**
